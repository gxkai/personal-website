This article will cover the basic concepts of Java, and you'll be done with it in thirty minutes.

Prerequisites

This article assumes that you have:

  • A way to run java code. This can be done using an IDE like IntelliJ Idea, a tool like BlueJ or an online playground like SoloLearn. Alternatively, you can directly run java files using the Java Development Kit.

Anatomy of a Java Program

Write this java code in your editor.

public class Program {
    public static void main(String[] args) {
        System.out.println("Hello, world");
    }
}

Don't worry about what this does, just execute the file and you should see Hello World printed on the screen.

Let's now run through this program. All java programs must be written in a class. We can define a class using class Classname. In the above program, we put public so that this class can be accessed by everyone. Everything in the pair of curly brackets ({}) after the class definition is part of the class.

Every class has a set of methods (sometimes called functions). The main() method in the above program is a special method. It tells Java that the code inside the main() method should be run when the program is called. Similar to the class, everything in the curly brackets ({}) after the method definition is part of the method. The code written inside the curly brackets gets executed when the method is run.

You'll learn more about methods later.

The System.out.println method is a function for printing stuff to the screen. This is how Hello World was printed on the screen. Remember this one, you'll use it a lot from now on.

Variables in Java

Variables are containers used for storing data. A variable can only hold one type of data in Java. This makes Java a strongly typed language, since you have to define the type of the variable when declaring it.

Here are some commonly used types of variables:

  • int - An integer. It can hold numbers between -232 and 232.

  • long - A longer integer. It can hold longer numbers.

  • float - A floating point. It can hold decimal numbers.

  • double - A longer decimal. It can hold longer floating points. This is generally used over the float datatype.

  • char - A character. It can hold a single character. It's value is ALWAYS enclosed in single quotes (').

  • boolean - An on/off switch. It can hold two states - true or false.

  • String - A string of text. It can hold any text of any length. It's value is ALWAYS enclosed in double quotes. The "Hello World" in the above program is a String

There are many many more types in java, but here are the most common ones.

To declare a variable, follow this syntax:

type variableName = value;

Never forget the semicolon (;)! This character is used to terminate a statement in java. It is always put after a code statement, EXCEPT when a block is being declared â€“ when there's a { or } at the end

Let's create a few variables in our program:

String firstName = "John";
String lastName = "Doe";
int age = 21;
boolean married = true;
char gender = 'M';
double gpa = 3.5;

Notice how the variable names are always ONE WORD? This is because in java, variables can not have spaces in their names. To differentiate between words in variables, we use a type of casing called camelCase. This means that every first letter of every word is capitalized EXCEPT the first letter of the first word.

You can print out the value of a variable by using the System.out.println() function:

System.out.println(firstName);
System.out.println(lastName);
System.out.println(age);

You should see John, Doe and 21 printed on your console on separate lines. Notice how there are NO QUOTES this time? This is because we're referring to a variable name, instead of declaring a string. See the code in the code block below and tell me what you think will be outputted:

System.out.println(firstName);
System.out.println("lastName");

The output will be John and lastName on separate lines, since firstName is referring to a variable, and "lastName" is creating a string with the value lastName.

Changing the value of variables

You are allowed to change the values of variables in java. This can be done using a similar syntax to declaring variables, but instead, you must omit the type.

int x = 10;
System.out.println(x); // Output: 10

x = 20;
System.out.println(x); // Output: 20

The code beginning with // are called comments. These are IGNORED by java, and are only visible to people who are directly viewing the file. This is useful for putting notes or in this case, writing the output of the following statement, for easier understanding.

You can also declare variable WITHOUT giving it a value, i.e. without initializing it.

int x; // Declare the variable

x = 10; // Give it a value

You can also set the value of a variable to another variable. This really only works with primitive datatypes. You'll learn more about primitive datatypes in the next section. To set the value of a variable to another variable, just put the name of the variable as the new value:

int a = 10;
int b = a; // b now has the value of a (10)

Final variables

Say you don't want to be able to change the value of variable. You can use the final keyword before declaring the variable. If you use final variables however, you HAVE to initialize it at the time of declaration. Here's an example:

final float pi = 3.14;

System.out.println(pi); // Output: 3.14

If you try changing the value of pi, you will get an error.

Declare many variables at once

You can declare multiple variables on one line IF they are the same type.

int x = 10, y = 20, z = 30;

String a, b, c; // Declaring without initializing

Data types and Type casting

Let's learn more about variable types or data types in java. There are two types of data types in java â€“ primitive and non-primitive types.

Primitive types

Primitive data types specify the size and type of the variable only. They have no additional methods and are built-in to java.

There are eight primitive datatypes in java.

Data Type Size Description
byte 1 byte Stores whole numbers from -128 to 127
short 2 bytes Stores whole numbers from -32,768 to 32,767
int 4 bytes Stores whole numbers from -2,147,483,648 to 2,147,483,647
long 8 bytes Stores whole numbers from -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807
float 4 bytes Stores fractional numbers. Sufficient for storing 6 to 7 decimal digits
double 8 bytes Stores fractional numbers. Sufficient for storing 15 decimal digits
boolean 1 bit Stores true or false values
char 2 bytes Stores a single character/letter or ASCII values

Non-Primitive Types

Non Primitive data types specify the type of the variable and provide methods and attributes to modify the type. Non-Primitive types can be created by the programmer, but creating primitive types is impossible. The String type is a non-primitive data type, since it allows you to modify its value using methods. We'll learn more about string methods later on.

Type Casting

It is possible to convert variables from one type to another. In this section, you'll learn how to convert variables of one primitive data type to another.

There are two types of type casts â€“ the one done by java automatically (Implicit casting) and the one done manually by us (Explicit casting).

Implicit casting is done from a datatype with a smaller size to a datatype of a larger size, and explicit casting is done the other way around. See the table in the section about primitive datatypes to see the sizes of different primitive types.

Let's do some type casting:

// Implicit
int anInt = 10; // Int has a size of 4 bytes
double aDouble = anInt; // Double has a size of 8 bytes, so it is bigger than int

System.out.println(anInt); // Output: 10
System.out.println(aDouble); // Output: 10.00

Notice how the .00 part of the double was automatically set for us. This is why implicit casting is only possible from smaller to larger types.

We can manually cast from a larger to a smaller type by using the type cast syntax. To cast a variable from type type1 to type type2:

type2 variable_name = (type2) variable_of_type1_type;

Let's see some explicit casting in action:

// Explicit
double aDouble = 3.14; // Double has a size of 8 bytes
int anInt = (int) aDouble; // Int has a size of 4 bytes, so it is smaller

System.out.println(aDouble); // Output: 3.14
System.out.println(anInt); // Output: 3

// Explicit casting also works the other way, but it is pointless
double anotherDouble = (double) anInt;
System.out.println(anotherDouble); // Output: 3.00

Wait! Where did the .14 go? Since int can't hold decimal figures, the 3.14 had to be rounded to 3 so that it can be stored in the int variable. When we converted int to double again, the .14 was lost.

Operators in Java

Operators are used to perform operations on variables and values. Operators can be used on variables, values, or a variable and a value.

There are six types of operators in java:

  • Arithmetic operators â€“ For performing mathematical operations

  • Concatenation operator â€“ For merging two strings together

  • Assignment operators â€“ For assigning values to variables

  • Comparison operators â€“ For comparing two variables or values

  • Logical operators â€“ For defining logic

  • Bitwise operators â€“ For performing operations on the bits and bytes of variables and values. We'll not be looking at bitwise operators in this article.

Arithmetic operators

There are 7 arithmetic operators in java.

Operator Name Description Example
+ Addition Adds together two values x + y
- Subtraction Subtracts one value from another x - y
* Multiplication Multiplies two values x * y
/ Division Divides one value by another x / y
% Modulus Divides two values and returns the reminder x % y
++ Increment Increases the value of a variable by 1 ++x
-- Decrement Decreases the value of a variable by 1 --x

The first five are pretty self explanatory. Let's take a look at the ++ and -- operators. They are of two types â€“ postfix and prefix. The main difference is that prefix FIRST increments/decrements the variable and then returns the value, and postfix FIRST returns the value and then increments/decrements the variable. Let's take a look at these operators:

int x = 10;

// Postfix
x++; // Increment 10 to 11
x--; // Decrement 11 to 10

// Prefix
++x; // Increment 10 to 11
--x; // Decrement 11 to 10

Well they do the same thing right? Well, not really. The main difference comes into light when you use the values returned by these operations.

int x = 10;
System.out.println(x++); // Output: 10
// The actual value of x is 11

x = 10; // Set x back to 10
System.out.println(++x); // Output: 11
// The actual value of x is 11

This happens because the postfix operator (x++) returns first and increments later, and the prefix operator (++x) increments first and returns later. In general you should always use the prefix operator to avoid hidden bugs.

Concatenation operator

This operator allows us to add/merge two strings together. It is represented by the plus sign (+).

String a = "Hello";
String b = " world";
String c = a + b; // c becomes: Hello world

The concatenation operator can also be used with a string and another primitive type value:

int x = 10;
System.out.println("x is: " + x); // Output: x is 10

Be sure to not get confused with the arithmetic addition operator and the string concatenation operator.

int anInt = 10;
String aString = "10"; // 10, in a string
System.out.println(anInt + 5); // Output: 15 (int)
System.out.println(aString + 5); // Output: 105 (String)

This happens because the concatenation operator converts all of its operands to strings first, and then does the operation while the addition operator only works with numbers.

Assignment operators

Assignment operators set values to variables. You've been using the = operator this whole time. That is the default assignment operator, and this operator is the only operator that works when the variable is not already defined.

There are other assignment operators, that are a combination of an arithmetic operator and the default assignment operator (=). Below are some examples:

int x = 10;
x += 5; // or x = x + 5
x -= 5; // or x = x - 5
x *= 5; // or x = x * 5
x /= 5; // or x = x / 5
x %= 5; // or x = x % 5

These assignment operators are meant merely for the ease of programming.

Comparison operators

These operators are used to compare two values. They only work with numbers.

Operator Name Example
== Equal to x == y
!= Not equal x != y
> Greater than x > y
< Less than x < y
>= Greater than or equal to x >= y
<= Less than or equal to x <= y

These operators always return a boolean.

boolean aBoolean = 5 > 3; // true
boolean anotherBoolean = 10 != 10; // false

Why == and why not =?

The equal to operator == has two == symbols because a single = is the assignment operator, and unlike the concatenation and addition operators, these two operators CAN clash, like in the example below:

boolean x = 10 == 10;

So be sure to double-check the correct operator!

Logical operators

Logical operators add logic to operations. The AND (&&) and the OR operators (||) are used to combine multiple conditional operators together. These operators return booleans based on their operands.

AND operator (&&):

Syntax: boolean1 && boolean2

This operator returns true if both boolean1 and boolean2 are true, and false otherwise.

Value of boolean1 Value of boolean2 Returned value
true true true
false true false
true false false
false false false

Example:

10 > 20 && 20 < 10; // true

OR operator (||)

Syntax: boolean1 || boolean2

This operator returns true if either boolean1 or boolean2 are true, and false otherwise.

Value of boolean1 Value of boolean2 Returned value
true true true
false true true
true false true
false false false

Example:

10 > 20 || 20 > 10; // true

NOT operator (!)

There is another logical operator called NOT. This one behaves differently from the other two. It reverses the boolean value given to it, meaning if true is passed to it, it returns false and vice-versa.

Syntax: !boolean1

Example:

System.out.println(!true); // Output: false

// Paranthesis can be used for operations that have spaces in them
System.out.println(!(3 > 5)); // Output: true

Conditional programming in java

Here comes the meat of programming â€“ conditional programming. Conditional programming is performing different actions based on different conditions. Say you wanted to perform a certain action when the value of a certain variable equals 10, or you wanted to run some code when a boolean is false, you can do that using java's two types of conditional statements - if and switch.

if, if..else and if..else if..else statements

if statements in java perform the defined action IF the boolean passed to it is true. Here's the basic syntax:

if (boolean1) {
    // Code that will be run if boolean1 is true
} else if (boolean2) {
    // Code that will be run if boolean2 is true AND the above boolean is false
} else {
    // Code that will be run if both the booleans above are false
}

// You can add as many else ifs as you want, or none at all, but else can only be added once

Let's see some examples:

Program to see if a user is above 18 years of age:

int age = 18;
if (age < 18) {
    System.out.println("You're under 18!");
} else {
    System.out.println("You're above 18!");
} // Output: You're above 18!

Program to print if a number is even or odd:

int num = 3;
if (num % 2 == 0) { // Even numbers are divisible by 2 - they give 0 remainder
    System.out.println("Even");
} else {
    System.out.println("Odd");
} // Output: Odd

Program to print if a number is negative, positive or zero:

int num = -5;
// If there are no brackets, the next statement and ONLY the next statement is considered
if (num < 0) System.out.println("Negative");
else if (num > 0) System.out.println("Positive");
else if (num == 0) System.out.println("Zero");
// We don't need an else, since one of the above booleans WILL be true.

// Output: Negative

Program to show a secret message if the user's name is admin:

String username = "not_admin";
if (username == "admin") System.out.println("You are the admin!");
System.out.println("Hello, " + username);
// Output: Hello, not_admin

switch...case statements

If your conditions are all using the == operator, and you have many many conditions, you can use the switch..case statement instead of if..else.

Here's the syntax:

switch(variable_to_be_checked) {
    case value1: // variable equals value1
        // code
        break; // Optional: Terminates the switch statement
    case value2: // variable equals value2
        // code
        break;
    default: // None of the above values matched (like else in if..else)
        // code
        break;
}

Here's an example program:

int channelNumber = 1;
switch(channelNumber) {
    case 1:
        System.out.println("Loading news channel");
        break;
    case 2:
        System.out.println("Loading sports channel");
        break;
    case 3:
        System.out.println("Loading cartoon channel");
        break;
    default:
        System.out.println("No channel found");
        break; // break is optional here, as it does nothing
}

What is break?

The break statement makes java exit out of the switch statement. Due to the nature of the switch statement, it will keep going through cases until it reaches the end, or encounters a break. That's why it is important to break out whenever there's a match, and the code has been executed, but this is also intentional, since it allows us to add aliases. Let's say I wanted to allow channel numbers 4 and 5 to match the news channel as well in the above program. Here's how the code will look:

int channelNumber = 1;
switch(channelNumber) {
    case 1:
    case 4:
    case 5:
        System.out.println("Loading news channel");
        break;
    case 2:
        System.out.println("Loading sports channel");
        break;
    case 3:
        System.out.println("Loading cartoon channel");
        break;
    default:
        System.out.println("No channel found");
        break; // break is optional here, as it does nothing
}

Since by default, switch keeps on going through cases until it is told to get out by the break keyword or reaches the end, we can use it to create fall-through cases to match more than one value.

Loops in java

In java, the code moves from top to bottom, once. If we want to repeat some code a certain number of times, we can use loops. There are two types of loops in java â€“ the while loop and the for loop. Let's take a look at these loops.

while loop

The while loop accepts a boolean condition, and executes the code within the {} as long as the condition is true.

Syntax:

while (boolean1) {
    // code
}

Example program to print out numbers from 1 to 5:

int i = 1;
while (i <= 5) {
    System.out.println(i);
    i++; // If this is omitted, the loop goes on forever!
}

The do..while loop

There is another kind of while loop: The do..while loop. The syntax of this is similar to that of the while loop. The only difference is that the code in this loop will execute ATLEAST once.

Syntax:

do {
    // code
} while (boolean1);
}

The for loop

The for loop is different. Instead of accepting just a condition like the while loop, it accepts three statements. This loop should be used when you know the number of times the loop should execute.

Syntax:

for (statement 1; statement 2; statement 3) {
    // code
}

Here's how the statements will be executed:

  • Statement 1 is executed ONCE before the execution of the loop. This statement should be used for declaring/initializing variables

  • Statement 2 is the condition for the loop.

  • Statement 3 is executed AFTER EVERY loop iteration. This statement should be used for changing the variables declared in statement 1.

We can convert the above while loop program for printing numbers from 1 to 5 to a for loop:

for (int i = 1; i <= 5; i++) {
    System.out.println(i);
}

You will notice that the statements used in both the programs are the same, except that in the for loop, the statements are in the loop.

break and continue

The break and continue statements are used in loops. You've already seen that the break keyword is used in switch to exit out of the switch statement. It can also be used in loops to exit out of the loop.

Example:

int i = 0;
while (i < 10) {
    if (i == 4) break;
    System.out.println(i);
    i++
}

Output of the above loop:

0
1
2
3

The loop will exit when the break statement is executed.

continue:

The continue statement is similar to the break statement, but instead of exiting the loop entirely, it starts the next iteration of the loop, meaning it puts the control back to the top.

Example:

for (int i = 0; i < 10; i++) {
    if (i == 4) continue;
    System.out.println(i);
}

Output of the above loop:

0
1
2
3
5
6
7
8
9

Since the continue statement transfers control back to the top of the loop, it is essential that the statement responsible for changing the value of the control variable is executed before calling continue;. This will prevent the loop from never stopping (becoming an infinite loop). That's why continue is usually used in for loops, since the increment statement always runs after every iteration.
